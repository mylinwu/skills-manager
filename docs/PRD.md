# Skills 可视化管理工具 - 产品需求文档

## 一、产品概述

基于 `skills` CLI 库的桌面端可视化管理工具，用于管理 AI Agent 技能包的安装、分发和更新。

## 二、技术栈

- **桌面框架**：Tauri
- **前端**：React + Tailwind CSS + Lucide React
- **状态管理**：Zustand
- **核心依赖**：skills CLI（通过 npx 调用）

## 三、功能模块详细说明

### 3.1 环境检测（初始化界面）

**检测项**：

- Node.js 是否安装
- skills 库是否可用（执行 `npx skills --version`）

**交互流程**：

1. 应用启动时自动检测
2. 检测未通过时显示：
   - 文字引导说明
   - 安装命令（可根据操作系统显示对应命令）
   - "复制命令"按钮
   - "重新检测"按钮
3. 检测通过后进入主界面

### 3.2 主界面结构

**左侧菜单**：

- 技能管理
- 技能分发
- 安装技能
- 设置

### 3.3 技能管理

#### 核心概念

**根技能目录**：

- macOS/Linux：`~/.agents/skills`
- Windows：`%USERPROFILE%\.agents\skills`

**多目录管理**：

- 用户可在设置中配置多个技能目录
- 技能以文件夹名称为唯一标识
- 同名技能去重规则：**后配置的目录优先**

#### 功能列表

**技能列表展示**：

- 显示所有技能的详细信息：
  - 技能名称（必须）
  - 描述（如果有）
  - 版本（如果有）
  - 作者（如果有）
  - 所在目录路径

**删除技能**：

- 点击删除按钮
- 弹窗确认，提示："删除技能会连带删除所有平台的软链接，确认删除吗？"
- 确认后：
  1. 删除根目录中的技能文件夹
  2. 自动删除所有平台中该技能的软链接
  3. 删除所有平台中该技能的复制副本

**更新技能**：

1. **检测更新**：
   - 执行 `npx skills check`
   - 显示有更新的技能列表

2. **更新操作**：
   - 更新单个技能：`npx skills update <技能名>`
   - 更新全部技能：`npx skills update`

3. **连带更新逻辑**：
   - 更新完成后，检测哪些平台使用"复制"方式分发了该技能
   - 弹窗提示："检测到 X 个平台使用复制方式分发了此技能，是否同步更新？"
   - 用户确认后，更新所有使用复制方式的平台副本

### 3.4 技能分发

#### 默认平台配置

| 平台代号 | macOS/Linux 路径 | Windows 路径 |
|---------|-----------------|-------------|
| claude-code | `~/.claude` | `%USERPROFILE%\.claude` |
| cursor | `~/.cursor` | `%USERPROFILE%\.cursor` |
| codex | `~/.codex` | `%USERPROFILE%\.codex` |
| opencode | `~/.config/opencode` | `%USERPROFILE%\.config\opencode` |
| amp | `~/.config/amp` | `%USERPROFILE%\.config\amp` |
| kilocode | `~/.kilocode` | `%USERPROFILE%\.kilocode` |
| roo | `~/.roo` | `%USERPROFILE%\.roo` |
| goose | `~/.config/goose` | `%USERPROFILE%\.config\goose` |
| gemini | `~/.gemini/antigravity` | `%USERPROFILE%\.gemini\antigravity` |

#### 界面交互

**平台选择**：

- 左侧显示平台列表
- 点击平台，右侧显示该平台的技能列表

**技能列表展示**：

- 合并显示：根技能目录 + 平台技能目录
- 每个技能显示状态标识：

| 状态 | 条件 | 显示组件 | 说明 |
|-----|------|---------|------|
| 已启用 | 根目录有 + 平台目录有 | Switch（开启状态） | 技能已分发到该平台 |
| 未启用 | 根目录有 + 平台目录无 | Switch（关闭状态） | 技能未分发到该平台 |
| 仅平台 | 根目录无 + 平台目录有 | 删除按钮 | 该技能只存在于平台目录 |
| 失效链接 | 软链接指向的源文件不存在 | "失效"标记 + 清理按钮 | 软链接已失效 |

#### 操作逻辑

**启用技能（Switch 从关闭到开启）**：

1. 读取该平台配置的分发方式
2. 根据分发方式执行：
   - **询问模式**：弹窗让用户选择"复制"或"软链接"
   - **复制模式**：直接复制技能文件夹到平台目录
   - **软链接模式**：创建符号链接到平台目录

**禁用技能（Switch 从开启到关闭）**：

1. 弹窗确认："确认禁用该技能吗？"
2. 确认后删除平台目录中的技能（文件夹或软链接）

**删除仅平台技能**：

1. 弹窗确认："该技能仅存在于此平台，确认删除吗？"
2. 确认后删除平台目录中的技能

**清理失效链接**：

1. 点击"清理"按钮
2. 删除失效的软链接

#### 软链接失效处理

**场景 1：通过技能管理删除**

- 自动连带删除所有平台的软链接和副本

**场景 2：用户从其他途径删除**

- 在技能分发页面标记为"失效"
- 显示清理按钮，用户手动触发清理

### 3.5 安装技能

#### 功能流程

1. **搜索技能**：
   - 输入框：输入技能名称或关键词
   - 点击"查找"按钮
   - 执行 `npx skills find <技能名>`
   - 展示搜索结果列表（名称、描述、版本等）

2. **安装技能**：
   - 点击某个技能的"安装"按钮
   - 执行 `npx skills add <技能名>`
   - **一定会安装到** `~/.agents/skills` 目录

3. **自动分发**：
   - 安装完成后，读取设置中的"默认分发平台列表"
   - 根据每个平台配置的分发方式，自动分发到对应平台
   - 如果平台配置为"询问"，则弹窗询问

### 3.6 设置

#### 6.1 技能目录配置

**功能**：

- 显示当前配置的所有技能目录
- 添加新目录：输入路径或浏览选择
- 删除目录：点击删除按钮
- 调整优先级：支持拖拽排序（列表底部优先级最高）
- 显示优先级说明："同名技能时，列表底部的目录优先"

#### 6.2 平台配置

**功能**：

- 显示所有平台配置列表
- 添加平台：
  - 平台名称
  - macOS/Linux 技能目录路径
  - Windows 技能目录路径
  - 分发方式：下拉选择
    - 询问（每次都询问）
    - 复制
    - 软链接
- 编辑平台：修改现有平台配置
- 删除平台：删除平台配置（不影响实际文件）

#### 6.3 默认分发平台列表

**功能**：

- 多选框列表，显示所有已配置的平台
- 勾选的平台会在"安装技能"后自动分发
- 说明文字："安装新技能后，会自动分发到勾选的平台"

## 四、技术实现要点

### 4.1 路径处理

- 使用 Tauri 的路径 API 处理跨平台路径
- 自动识别操作系统并使用对应的路径格式
- 处理 `~` 和 `%USERPROFILE%` 等环境变量

### 4.2 命令执行

- 通过 Tauri 的 Command API 在后端执行 npx 命令
- 捕获命令输出和错误信息
- 显示执行进度和结果

### 4.3 文件操作

- 复制：使用文件系统 API 递归复制文件夹
- 软链接：使用符号链接 API（注意 Windows 需要管理员权限或开发者模式）
- 删除：递归删除文件夹或删除软链接

### 4.4 状态管理

- 使用 Zustand 管理：
  - 技能列表
  - 平台配置
  - 设置项
  - UI 状态（加载中、错误提示等）

---

## 五、用户体验细节

### 5.1 加载状态

- 所有异步操作显示加载指示器
- 长时间操作显示进度条（如果可能）

### 5.2 错误处理

- 命令执行失败时显示错误信息
- 文件操作失败时提示具体原因
- 提供重试选项

### 5.3 确认弹窗

- 所有删除操作都需要确认
- 确认弹窗清晰说明操作后果

### 5.4 成功反馈

- 操作成功后显示 Toast 提示
- 自动刷新相关列表
