# Security Policy

## Supported Versions

当前仓库优先处理最新主线代码中的安全问题。历史版本默认不提供长期安全维护承诺。

## Reporting a Vulnerability

如果你发现了安全漏洞，请不要直接公开提 Issue。

建议通过私下渠道联系维护者，并尽量提供：

- 漏洞影响范围
- 复现步骤
- 受影响环境
- 可能的修复建议或缓解方式

在仓库公开早期，如果还没有单独的安全邮箱或表单，维护者会先通过私下沟通完成确认与修复，再决定披露时间。

## Response Expectations

- 我们会尽快确认问题是否可以复现
- 在问题确认后优先处理高风险漏洞
- 修复发布前，不会公开敏感利用细节
